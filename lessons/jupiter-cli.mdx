## Powering Agentic Trading with Jupiter CLI

Almost everyone interacts with Jupiter through a browser or Jupiter Mobile. You connect a wallet, you read the screen, and you approve each transaction by hand. It is convenient, safe, and exactly what most people should use. A human looks at a price and decides to click.

But there is a different kind of user that is not a person at all. It is software: a script running on a schedule, or an AI agent reasoning about the market and acting on its own. These do not read screens or click buttons. They need a way to ask for prices and execute trades as plain text. **Jupiter Command Line Interface (CLI) is the surface they trade through**, and these are typically built by power users, professional traders, and developers.

If you are not building that kind of system, the [Jupiter app](https://jup.ag) is the better tool for you. But if you are building an agentic system that needs speed, automation, and composability, Jupiter CLI gives it direct, programmatic access to the full Jupiter product suite.

### CLI - the Start of True Automation

A Command-Line Interface (CLI) has no buttons, charts, or pop-ups. You issue commands as text and you get text back. That sounds primitive, but it is precisely what makes it the right shape for automation.

Every Jupiter CLI command is non-interactive: it runs start to finish without ever pausing to ask a human for confirmation. That single property is what lets a script or an agent drive Jupiter unattended. The browser is designed so a person can carefully review actions. The CLI is designed so a program never has to wait for one.

- **Speed**. Commands execute immediately. There are no loading screens, no pop-ups, no confirmation dialogs.
- **Scriptability**. Because every command is plain text, you can write scripts that execute dozens of trades, monitor positions, or rebalance portfolios automatically on a schedule or in response to market conditions.
- **Composability**. CLI outputs can be piped directly into other DeFi tools, all of which can be automated via more commands.

Through it, an agent can reach the full Jupiter product suite: swap tokens on Spot, earn yield on Lend, trade perps, and take positions in Prediction Markets, all from the same interface.

A few examples of what CLI commands look like in practice:

```bash
# Generate a new keypair
jup keys add mykey

# Swap 1 SOL to USDC
jup spot swap --from SOL --to USDC --amount 1

# Open a leveraged long position
jup perps open --asset SOL --side long --amount 10 --input USDC --leverage 3

# Deposit USDC into Jupiter Lend
jup lend earn deposit --token USDC --amount 100
```

### Understanding JSON Output Mode

By default, most command-line tools return human-readable text: formatted columns, color-coded status messages, and helpful labels. That presentation is great for humans, but if you want to integrate with other DeFi products directly, it is difficult for other machines and protocols to reliably interpret.

Therefore, Jupiter CLI relies on JSON output mode, an easily configured option. By enabling JSON output mode, every command returns structured data instead of readable formatted text. This is important to allow further machine-enabled manipulation of the data.

Think of it this way: a restaurant delivers a finished plate, arranged and garnished, ready to eat. A JSON response delivers the raw ingredients: each one labeled, separated, and ready to be used however you choose. You can then easily route that data into a spreadsheet, a Telegram bot, an AI agent, or a custom dashboard of your choice.

### Wallet Management

The CLI manages wallets through keypairs stored locally on your machine. You can generate a new keypair directly from the CLI, import an existing Solana CLI keypair, or load one from a seed phrase. Your private key never leaves your machine; it is only stored locally and not on any server.

That is the trade-off of going agentic. There is no extension popping up to approve each transaction, because the whole point is that no human is in the loop. The agent acts with the authority of a key that lives on your disk, so securing that key and defining what the agent is permitted to do is your responsibility.

### Fast, with No Guardrails

This is the most important concept in the entire lesson. The browser interface has guardrails: transaction simulation, confirmation screens, and clear summaries of what will happen before you approve. The CLI has none of those by default. When you execute a command, it signs and executes the transaction immediately.

A wrong token ticker, an incorrect amount, or a mistyped slippage value will be executed onchain before you can intervene. There is no undo button in the blockchain world. It's highly recommended to run every new command type with a small amount first, verify the output matches your expectations, then scale up. Treat your first few CLI sessions as a testing environment.

> **Note:** Jupiter CLI is in pre-V1 / early alpha. Breaking changes may be introduced without notice. It is not recommended to build production-critical systems on top of it at this stage.
